TO IMPROVE

1. Move from Waterfall to LangGraph Orchestration (Mandatory)
You should absolutely migrate the entire orchestration layer to LangGraph.

A linear script treats errors as fatal exceptions (throw new Error). LangGraph treats failures as state transitions (edges). When validation fails, LangGraph shifts the state backward to a correction node instead of crashing the process. This shift allows you to manage long-running async steps, implement human-in-the-loop checkpoints, and cleanly scale execution state.

2. Implement the Three Core Self-Healing Loops
Instead of a single massive feedback loop, we should break self-healing down into three distinct, specialized sub-graphs inside LangGraph:

[Planner] ──> [Decomposer] ──> [File Generator] ──> [AST Check] ──> [Linter/Typecheck] ──> [Review Gate]
                                    ▲                      │                 │                  │
                                    │─── (Fix Syntax) ─────┘                 │                  │
                                    │─── (Fix Type Errors) ──────────────────┘                  │
                                    └─── (Address Feedback) ────────────────────────────────────┘
Loop A: The Compiler Feedback Loop (Syntax-Level)
Trigger: runLocalASTCheck fails.

Mechanism: Do not invoke a heavy LLM or call your full summarizer again. Route the raw file contents along with the exact line-number error directly back to a specialized SyntaxFixerNode.

Goal: Ensure the code is structurally parseable before any heavy compute is spent testing it.

Loop B: The Type & Compliance Loop (System-Level)
Trigger: runProjectVerification fails (Linter or Typecheck crashes).

Mechanism: Feed the linter/TSC stdout error string into your context window. Give the agent a strict budget (e.g., maximum 3 attempts) to fix missing imports, incorrect types, or variable mismatches across the files it just touched.

Loop C: The Architectural Review Loop (Semantic-Level)
Trigger: ReviewerAgent returns requires_changes or reject.

Mechanism: Feed the actionableFeedback array generated by your reviewer back into the DecomposerAgent or FileGeneratorAgent. The coding agent receives the feedback as a direct instruction: "Your previous attempt was rejected for XYZ reasons. Rewrite the patch to address these specific issues."

3. Dynamic Context Scaling & Token Management
Your current RetrievalEngine pulls a static topK = 6 files and reads their entire text contents into the LLM context window. On a medium-sized codebase, this will result in massive context bloat, high API costs, and degraded model performance due to the "lost in the middle" phenomenon.

The Improvement:
Two-Tier Retrieval: First, use the hybrid indexer to find relevant files. Second, instead of reading the whole file, read only the specific symbol boundaries (the exact class or function text blocks) that match the ticket scope, using ts-morph's structural locations.

Dynamic Budgeting: Build a context window pruner. If the combined size of the retrieved files exceeds a predefined token threshold, the system should automatically swap out complete file reads for their short generated summaries instead.

4. Sandbox Execution Environment (Security & Runtime Validation)
Right now, you are running execSync("npm run typecheck") directly on your host machine workspace. This approach introduces two critical risks:

Security: If the agent hallucinates or generates malicious code (e.g., an unintended fs.rmdirSync), it executes natively with your machine's user permissions.

Runtime Testing: You cannot easily execute the repository's unit tests (npm run test) to verify if the agent actually broke business logic, because running local test suites can pollute your host system or leave dirty databases behind.

The Improvement:
Move code execution, linting, typechecking, and test suite execution inside an isolated Docker Container Sandbox or a secure micro-VM.

The workflow service writes files locally, mounts them to an isolated container, triggers execution, captures strings from stdout/stderr, and terminates the environment safely.
